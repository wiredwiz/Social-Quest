SocialQuest Changelog
=====================

Version 2.18.2 (April 2026)
- Bug fix: "Everyone has completed" banner failed to fire on Retail when party members
  held variant questIDs for the same logical quest. checkAllCompleted now falls back to
  direct questID match when title resolution fails, and finds the local player's variant
  quest by title scan when AQL:GetQuest returns nil for the triggering questID.
  Verbose debug logging added to checkAllCompleted (fires when debug.enabled).

Version 2.18.1 (April 2026)
- Feature: No-objective quest status display. Quests with no numeric X/Y objectives
  (travel, talk-to-NPC, exploration) now show per-player status in Party and Shared tabs
  using a two-column layout: player name left, status text left-aligned at bar start
  position. Three states: "Finished" (quest turned in, green), "Complete" (objectives
  met not yet turned in, green), "In Progress" (no objectives, not yet done, dimmed).
  Mine tab title row gains an `(In Progress)` badge at lowest priority (after `(Complete)`
  and `(Group)`).
- Refactor: `hasCompleted` and `isComplete` player rows now use `renderStatusRow` for
  consistent two-column layout. Single-string fallback preserved when `nameColumnWidth`
  is nil.
- i18n: new locale keys `Finished`, `In Progress`, `(In Progress)` in all 12 locales.
  Removed `%s FINISHED` format-string key (superseded by standalone `Finished`).

Version 2.15.0 (March 2026)
- Added a share button and sharing eligability detection to the party tab.
  Now it better detects who is eligable for a quest and if they aren't, tells
  you why.  The share button will share the quest with the party as you would
  expect
- Added a new "Shareable" filter key for the advanced quest list filtering.

Version 2.12.33 (March 2026)
- Removed flight path discovery feature. The TBC Classic taxi APIs only return
  data while the flight map UI is open, not when the player talks to a flight
  master. Since the discovery occurs at NPC gossip time but the API data is
  unavailable until the player explicitly opens the map, reliable detection is
  not possible. The feature has been removed entirely.

Version 2.12.31 (March 2026)
- Bug fix: zone auto-filter now shows the correct name in starter areas such as
  Northshire Abbey. The filter now prefers GetSubZoneText() (e.g. "Northshire
  Valley") over GetRealZoneText() (e.g. "Elwynn Forest") when a sub-zone name
  is available. The instance filter branch is unchanged.
- Feature: Mine tab quest objectives now render as progress bars matching the
  Party and Shared tabs, instead of plain colored text.

Version 2.12.18 (March 2026)
- Feature: Advanced filter language. The search bar now accepts structured filter
  expressions entered by pressing Enter (e.g. level>=60, zone=Elwynn|Deadmines,
  status=incomplete, player=Thad). Active filters are stored persistently and
  displayed as dismissible labels in the header. Multiple filters AND together with
  each other, the search text, and the auto-zone filter. A [?] button opens a
  movable syntax reference window. New modules: FilterParser, FilterState,
  HeaderLabel.

Version 2.12.17 (March 2026)
- Bug fix: quests received via SQ_INIT from other players were always shown as
  complete/failed due to integer-to-boolean conversion not being applied to the
  wire payload. isComplete, isFailed, and objective isFinished flags are now
  correctly converted on receipt.

Version 2.12.16 (March 2026)
- Bug fix: Questie bridge quest data did not arrive after /reload in some cases.
  Added a third request attempt at t+10s, by which point the party API is always
  stable and Questie's listener is reliably registered.

Version 2.12.15 (March 2026)
- Added debug logging to QuestieBridge Enable, request scheduling, stub creation,
  and hydration callback to aid diagnosing Questie bridge issues.

Version 2.12.14 (March 2026)
- Bug fix: Questie player's quests never appeared after /reload even after 30+
  seconds. On reload, party member unit names are unavailable during PLAYER_LOGIN
  so stubs were never created. New _EnsurePartyStubs() method creates missing
  stubs before each hydration poll, when unit names are reliably available.

Version 2.12.13 (March 2026)
- Bug fix: Questie quest request sent at t+1s on reload arrived before Questie's
  own message listener was registered (QuestieComms initializes up to 3s after
  load). Added a second request at t+5s to ensure it is received.

Version 2.12.12 (March 2026)
- Bug fix: Questie bridge did not request quest data on /reload because the
  party roster is not populated at PLAYER_LOGIN. Now requests directly from
  QuestieBridge:Enable(), which fires reliably on the first GROUP_ROSTER_UPDATE.

Version 2.12.11 (March 2026)
- Bug fix: SQ window restored to screen center after reload instead of its last
  position. Window position and size are now saved and restored across sessions
  and loading screens.

Version 2.12.10 (March 2026)
- Bug fix: objective progress bars for Questie bridge players showed the local
  player's count instead of the remote player's. The count is now stripped from
  the local objective text and re-appended with the remote player's values.

Version 2.12.9 (March 2026)
- Bug fix: Questie bridge quests never appeared on member join. The RegisterTooltip
  hook used to detect incoming data does not fire for V2 full-log packets.
  Switched to polling remoteQuestLogs directly at t+4s and t+8s after sending
  the quest log request.

Version 2.12.8 (March 2026)
- Refactor: replaced two separate Questie packet hooks (V1 and V2) with a single
  stable hook on QuestieComms.data:RegisterTooltip, which is called by all
  packet-insertion paths and passes questId directly.

Version 2.12.7 (March 2026)
- Bug fix: (superseded by 2.12.8) Hooked unstable V2 packet function name.

Version 2.12.6 (March 2026)
- Bug fix: (superseded by 2.12.8) Hooked wrong V2 packet function.

Version 2.12.5 (March 2026)
- Bug fix: Questie bridge player names with realm suffix did not match PlayerQuests
  entries stored under short names. Added name normalization that tries the full
  name first then falls back to the short name.

Version 2.12.4 (March 2026)
- Bug fix: an error in SocialQuestComm:OnMemberJoined was stopping execution
  before the Questie bridge join handler ran. Moved BridgeRegistry:OnMemberJoined
  before the comm call so bridge initialization cannot be blocked by comm errors.

Version 2.12.3 (March 2026)
- Bug fix: Questie player joining an existing group never had their quests appear.
  Questie does not broadcast on roster update, so SocialQuest now sends the
  QC_ID_REQUEST_FULL_QUESTLIST message directly when a member joins.

Version 2.12.2 (March 2026)
- Party and Shared tabs now refresh immediately when the group roster changes
  (member joins or leaves).

Version 2.12.1 (March 2026)
- Bug fix: Questie bridge live objective updates used abbreviated field names from
  the raw packet instead of the full names expected by the entry builder.
- Added deferred re-hydration on first contact so all packets in a comm batch
  land in remoteQuestLogs before the snapshot is taken.

Version 2.12.0 (March 2026)
- Bug fix: QuestieBridge referenced QuestieComms as a global, which only exists
  in Questie debug mode. Now resolves via QuestieLoader:ImportModule().

Version 2.11.1 (March 2026)
- Expand/collapse all buttons moved from scrollable content to the fixed header
  row, so they remain visible regardless of scroll position.

Version 2.11.0 (March 2026)
- Search bar: a persistent search box appears below the tab strip and filters all
  three tabs by quest or chain title (case-insensitive). Search text is shared
  across tabs and preserved across loading screens.
- Zone/instance filter label moved from scrollable content into the fixed header
  strip below the search bar.

Version 2.10.4 (March 2026)
- Bug fix: zone/instance filter did not update when a flight path ended in a
  different zone. PLAYER_CONTROL_GAINED now resets the filter when the taxi
  system releases the player at the destination.

Version 2.10.3 (March 2026)
- Feature: the SQ window now reopens automatically after loading screens
  (hearthstone, portals, instance entry/exit, /reload) if it was open before
  the transition.

Version 2.10.2 (March 2026)
- Filter label now prefixed with "Filter: " across all 12 locales.
- Bug fix: zone filter was not updating when crossing an overland zone border.
  ZONE_CHANGED_NEW_AREA now resets the filter for seamless zone transitions.

Version 2.10.1 (March 2026)
- Progress bar polish: replaced flat colored bars with WoW-native StatusBar
  widgets using the standard UI-StatusBar fill texture and a CastingBar border
  overlay. Objective text forced white with drop shadow; WoW color escapes
  stripped. Colorblind sky-blue for completed objectives unchanged.

Version 2.10.0 (March 2026)
- Progress Bars: objective rows in the Party and Shared tabs render as inline
  progress bars filling proportionally to numFulfilled/numRequired. Player names
  in a left-aligned column so all bars line up for at-a-glance comparison.
  Colorblind mode respected. Falls back to plain text when numeric data is absent.

Version 2.9.0 (March 2026)
- Zone & Instance Auto-Filter: Party and Shared tabs can optionally filter to the
  current dungeon/raid instance or open-world zone. New UI/WindowFilter.lua module
  owns all filter state. Two new toggles in /sq config: "Auto-filter to current
  instance" (default ON) and "Auto-filter to current zone" (default OFF). Filter
  resets on zone change, window close, or settings toggle.

Version 2.8.2 (March 2026)
- Bug fix: eliminated false "objective completed again" and false regression
  announcements when splitting or merging bag stacks of quest collectible items.
  Root fix in AQL (CursorHasItem() guard in handleQuestLogUpdate()) prevents
  false REGRESSED/PROGRESSED/COMPLETED events from ever firing. Removed the
  pendingRegressions AceTimer debounce from SocialQuest.lua — it was a workaround
  for those now-suppressed events and is no longer needed.

Version 2.8.1 (March 2026)
- Bug fix: clicking a quest title in the SQ window when the quest log was open,
  the quest selected, but its zone was collapsed caused the log to close instead
  of expanding the zone. Fixed by gating the close-toggle on the quest being
  visible (zone not collapsed) using AQL:GetQuestLogIndex().
- Bug fix: regression banner still appeared when splitting a quest item stack.
  Debounce window in OnObjectiveRegressed increased from 0.5 s to 2 s.

Version 2.8.0 (March 2026)
- Questie Bridge: hooks Questie's QuestieComms layer to populate PlayerQuests
  for party members who have Questie but not SocialQuest.
- Added SocialQuest.DataProviders and SocialQuest.EventTypes constant tables.
- Added dataProvider field to all PlayerQuests entries.
- New modules: Core/BridgeRegistry.lua and Core/QuestieBridge.lua.
- RowFactory appends the bridge's name tag icon for non-SQ data sources.

Version 2.7.0 (March 2026)
- Added Core/WowAPI.lua and Core/WowUI.lua abstraction modules. All direct WoW
  game-state and data API calls (GetTime, IsInRaid, IsInGroup, IsInGuild, UnitName,
  UnitFullName, UnitRace, UnitFactionGroup, GetNumGroupMembers, GetRaidRosterInfo,
  SendChatMessage, C_FriendList.*, C_Timer.After, GetTaxiNodeInfo) now route through
  SocialQuestWowAPI. Volatile UI primitives (RaidNotice_AddMessage, PanelTemplates_*,
  DEFAULT_CHAT_FRAME:AddMessage) route through SocialQuestWowUI. Quest and quest-log
  API calls replaced with AQL public API equivalents. Prepares SocialQuest to support
  multiple WoW interface versions without scattered direct API usage.

Version 2.6.0 (March 2026)
- Bug fix: chain header label in the Mine tab showed the title of whichever quest step
  the player was currently on, rather than a consistent name for the chain. Because WoW
  has no native chain-name concept, SocialQuest now resolves the chain label from step 1's
  quest title (via AQL:GetQuestInfo on the chainID, which is always the step-1 questID).
  The chain name is now stable regardless of how far into the chain the player is.
- Added addon logo for addon screen
- Added minimap button toggle to general configuration tab.

Version 2.5.1 (March 2026)
- Changed "Everyone has completed" banner color from dark purple (#9900E6) to hot
  magenta (#FF00CC) for better contrast against WoW's outdoor and dungeon environments.
  Colorblind color (Okabe-Ito blue #0072B2) unchanged.

- Bug fix: {rt1} displayed as literal text in the "Test Chat Link" chat preview. Added
  gsub in displayChatPreview to convert {rt1}–{rt8} to WoW texture escape sequences
  before passing to DEFAULT_CHAT_FRAME:AddMessage (SendChatMessage handles the
  substitution natively; AddMessage does not).
- Bug fix: Force Resync button did not automatically re-enable after its 30-second
  cooldown. Scheduled an AceConfigRegistry NotifyChange call 30 seconds after each
  press so the button re-enables without requiring a config panel reopen.

Version 2.5.0 (March 2026)
- Quest language consistency: "Complete" now means all objectives are filled (matching WoW's
  own quest log label). "Turned in" means the quest was delivered to the NPC. Updated banner
  text, outbound chat messages, options toggle labels, and debug test button names across all
  12 locale files. Non-English translations use natural player vocabulary for each language
  rather than literal translations (e.g. German "abgegeben", French "rendu", Korean "반납",
  Chinese "交任务").

Version 2.4.0 (March 2026)
- Follow banner notifications: OnFollowStart and OnFollowStop now display a
  banner in addition to the existing chat message. Uses new follow color key
  (warm tan / Okabe-Ito yellow in colorblind mode). Added TestFollowNotification
  debug function and corresponding debug panel button.

Version 2.3.7 (March 2026)
- Bug fix: six unescaped double-quote characters inside Chinese string values in
  Locales/zhCN.lua caused Lua syntax errors at load time. Escaped in the

  descriptions. zhTW was unaffected.
